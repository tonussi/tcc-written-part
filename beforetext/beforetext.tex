% Replication is a widely adopted fault tolerance strategy to achieve a minimum acceptable degree of availability, of the services. A technique widely used is State Machine Replication, where each replica starts from the same initial state, receives the same requests from the clients and processes the requests deterministically, ensuring that the replicas evolve equally through time. In this way, the occurrence of failures in a limited number of replicas does not compromise the functioning of the service. Recently, microservices-based architectures have gained popularity due to the modular programming model and also due to the support of container orchestration platforms, which offer automatic and efficient management of resources required by container orchestration platforms. This work presents a proposal for the implementation of State Machine Replication aimed at microservices architectures. The goal is to provide the replication functionality transparently to the user, so that the details of replication and fault tolerance do not need to be explicitly addressed by developers. This work proposal will extend a work initiated by the group, which proposes an architecture for a message ordering service based on interceptors. To improve the service, we intend to implement new communication and message ordering protocols, in addition to investigating resources to improve performance in ordering and delivering requests to replicas, such as the use of batches.
